### üåä Propagaci√≥n de Eventos (Bubbling y Capturing)

```javascript
document.querySelector('.exterior').addEventListener(
  'click',
  () => {
    console.log('Exterior');
  },
  true
); // Capturing
```

1. **Flujo de eventos en el DOM**  
   Cuando haces clic en un elemento, el navegador sigue tres fases:

   1. **Capturing (captura):** El evento viaja desde el `document` hasta el elemento clicado, **de arriba hacia abajo**.
   2. **Target (objetivo):** Se ejecutan los listeners en el elemento donde ocurri√≥ el clic.
   3. **Bubbling (burbujeo):** El evento sube desde el elemento clicado hasta `document`, **de abajo hacia arriba**.

2. **Tercer par√°metro de `addEventListener`**

   - Si es `false` (por defecto), el listener se ejecuta en la fase de **burbujeo**.
   - Si es `true`, el listener se ejecuta en la fase de **captura**.

3. **¬øQu√© hace el c√≥digo?**

   - Busca el elemento con clase `.exterior`.
   - Escucha el evento `click`.
   - Gracias al `true`, el `console.log('Exterior')` se ejecuta **mientras el evento baja (captura)**, antes de que llegue al objetivo.

4. **Ejemplo mental**  
   Si tienes:

   ```html
   <div class="exterior">
     <div class="interior">
       <button>Click aqu√≠</button>
     </div>
   </div>
   ```

   ```javascript
   document.querySelector('.exterior').addEventListener(
     'click',
     () => {
       console.log('Exterior');
     },
     true
   ); // Capturing

   document.querySelector('.interior').addEventListener('click', (e) => {
     console.log('Interior');
     // e.stopPropagation(); // Detener la propagaci√≥n
   });
   ```

Y pones listeners en cada capa (algunos con `true`, otros con `false`), el orden de ejecuci√≥n cambiar√°:

- Con `true`, el listener de `.exterior` corre **primero**.
- Con `false`, correr√≠a **despu√©s** de que el evento burbujee desde el bot√≥n hacia arriba.

üéØ En resumen: el `true` hace que el listener se ejecute **al principio del flujo del evento** (fase de captura), antes que otros listeners en burbujeo.

## ‚ùì Preguntas Frecuentes

### ¬øCu√°l es la diferencia entre `textContent` e `innerHTML`?

- `textContent` maneja el contenido como texto plano
- `innerHTML` interpreta el contenido como HTML (puede ser vulnerable a XSS si no se maneja con cuidado)

### ¬øQu√© es la delegaci√≥n de eventos?

Es una t√©cnica que aprovecha la propagaci√≥n de eventos para manejar eventos en m√∫ltiples elementos usando un solo manejador en un elemento padre.

### ¬øC√≥mo evito el comportamiento por defecto de un evento?

### ¬øCu√°l es la diferencia entre `window.onload` y `DOMContentLoaded`?

- `window.onload` espera a que se carguen todos los recursos (im√°genes, estilos, etc.)
- `DOMContentLoaded` se dispara cuando el HTML est√° completamente cargado y parseado, sin esperar recursos externos
